Branching & Merging

Branch memungkinkan kalian bereksperimen atau mengerjakan fitur tanpa mengganggu kode utama yang sudah stabil.

Di modul ini
  1. Apa itu branch
  2. Membuat & berpindah branch
  3. Menggabungkan branch (merge)
  4. Menyelesaikan merge conflict

Apa itu Branch?

Secara teknis, branch hanyalah pointer bergerak yang menunjuk ke commit tertentu. Karena hanya pointer (bukan salinan seluruh file), membuat branch di Git sangat ringan dan instan — beda dengan sistem lama yang harus menyalin seluruh proyek.

Kenapa pakai branch?

  • Mengerjakan fitur baru tanpa merusak branch main yang stabil.
  • Beberapa orang bisa mengerjakan fitur berbeda secara bersamaan.
  • Bisa membuang eksperimen yang gagal tanpa efek samping — cukup hapus branch-nya.
ilustrasi
main:     A---B---C---F---G
                \         /
feature:         D-------E

Branch feature dibuat dari commit B, berkembang lewat commit D dan E, lalu digabung kembali ke main pada commit G.

Membuat & Berpindah Branch

bash 
git branch                       # lihat daftar branch, tanda * = branch aktif
git branch nama-fitur             # buat branch baru (tanpa berpindah)
git switch nama-fitur             # berpindah ke branch tersebut
git switch -c nama-fitur          # buat sekaligus berpindah (cara modern)
git branch -d nama-fitur          # hapus branch yang sudah selesai di-merge
📌
git switch vs git checkout

git checkout -b nama-fitur adalah cara lama untuk membuat+berpindah branch, dan masih banyak dipakai. git switch -c adalah perintah modern (Git 2.23+) yang lebih fokus dan tidak ambigu, karena checkout dulunya juga dipakai untuk hal lain (mengembalikan file). Keduanya valid — pakai yang paling nyaman.

Menggabungkan Branch (Merge)

Setelah fitur di branch selesai dan diuji, gabungkan kembali ke main:

bash 
git switch main            # pindah ke branch tujuan
git merge nama-fitur       # gabungkan branch fitur ke main

Ada dua jenis merge:

JenisKapan terjadi
Fast-forward Branch main tidak punya commit baru sejak branch fitur dibuat — Git cukup menggeser pointer main ke depan, tanpa commit merge baru.
3-way merge Kedua branch sama-sama punya commit baru — Git membuat "merge commit" baru yang menggabungkan riwayat keduanya.

Menyelesaikan Merge Conflict

Conflict terjadi ketika dua branch mengubah baris yang sama pada file yang sama, dan Git tidak bisa memutuskan sendiri versi mana yang benar. Ini normal dan bukan tanda kesalahan — justru bagian penting dari kerja tim yang perlu kalian kuasai.

Saat conflict terjadi, git status akan menandai file bermasalah, dan isi filenya akan terlihat seperti ini:

contoh isi file saat conflict
<<<<<<< HEAD
<h1>Selamat Datang di Toko Kami</h1>
=======
<h1>Selamat Datang di Toko Online</h1>
>>>>>>> nama-fitur

Langkah menyelesaikannya:

  1. Buka file yang konflik di editor.
  2. Putuskan versi mana yang dipakai (atau gabungkan keduanya), lalu hapus semua marker <<<<<<<, =======, >>>>>>>.
  3. Simpan file, lalu stage dan commit seperti biasa:
bash 
git add nama-file-yang-konflik
git commit -m "Selesaikan konflik merge nama-fitur ke main"
⚠️
Kalau ragu, batalkan dulu

Sedang di tengah merge dan bingung? Jalankan git merge --abort untuk membatalkan dan kembali ke kondisi sebelum merge dimulai. Lebih aman daripada asal commit.

Memvisualisasikan Branch

bash 
git log --oneline --graph --all

Jalankan ini kapan pun kalian bingung dengan kondisi branch saat ini — sangat membantu membangun intuisi visual tentang apa yang sedang terjadi.
