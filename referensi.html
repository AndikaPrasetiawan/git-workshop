Cheat Sheet Git

Kumpulan perintah yang paling sering dipakai sehari-hari, dikelompokkan per kategori. Bookmark halaman ini.

Konfigurasi

PerintahFungsi
git config --global user.name "Nama"Atur nama untuk commit
git config --global user.email "email"Atur email untuk commit
git config --listLihat semua konfigurasi aktif

Dasar

PerintahFungsi
git initMulai repository baru di folder saat ini
git statusLihat kondisi working directory & staging area
git add <file>Stage file tertentu
git add .Stage semua perubahan
git commit -m "pesan"Simpan snapshot dari staging area
git commit -am "pesan"Stage file yang sudah dilacak + commit sekaligus
git log --onelineRiwayat commit versi ringkas
git diffLihat perubahan yang belum di-stage
git diff --stagedLihat perubahan yang sudah di-stage
git show <hash>Lihat detail satu commit

Branching & Merging

PerintahFungsi
git branchLihat daftar branch lokal
git branch <nama>Buat branch baru
git switch <nama>Pindah ke branch tertentu
git switch -c <nama>Buat sekaligus pindah ke branch baru
git branch -d <nama>Hapus branch yang sudah di-merge
git merge <nama>Gabungkan branch ke branch aktif saat ini
git merge --abortBatalkan merge yang sedang konflik
git log --oneline --graph --allVisualisasi riwayat semua branch

Remote & GitHub

PerintahFungsi
git clone <url>Unduh repo dari remote beserta riwayatnya
git remote add origin <url>Hubungkan repo lokal ke remote
git remote -vLihat daftar remote yang terhubung
git push -u origin mainPush pertama kali + set upstream branch
git pushKirim commit lokal ke remote
git fetchUnduh perubahan remote tanpa merge
git pullUnduh + merge perubahan remote (fetch + merge)

Undo & Perbaikan

PerintahFungsi
git restore <file>Buang perubahan pada file (belum di-stage), kembalikan ke versi terakhir commit
git restore --staged <file>Batalkan staging (unstage), perubahan file tetap ada
git commit --amendPerbaiki pesan/isi commit terakhir (sebelum di-push)
git revert <hash>Buat commit baru yang membatalkan efek commit tertentu (aman untuk riwayat yang sudah di-push)
git reset --soft <hash>Mundur ke commit tertentu, perubahan tetap di staging area
git reset --hard <hash>Mundur ke commit tertentu, buang semua perubahan setelahnya (hati-hati, destruktif)

Lainnya yang Sering Dipakai

PerintahFungsi
git stashSimpan sementara perubahan yang belum siap di-commit, kembalikan working directory ke kondisi bersih
git stash popKembalikan perubahan terakhir yang di-stash
git rm <file>Hapus file dari working directory sekaligus dari tracking Git
git rm --cached <file>Berhenti melacak file, tapi file tetap ada di komputer (dipakai bareng .gitignore)
git mv <lama> <baru>Rename/pindahkan file dengan tetap tercatat di riwayat
git blame <file>Lihat siapa mengubah tiap baris file, dan di commit mana
