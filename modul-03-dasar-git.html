Perintah Dasar Git

Ini modul inti — siklus init → add → commit → log yang akan kalian ulangi ratusan kali dalam karier ngoding.

Di modul ini
  1. git init
  2. git status
  3. git add
  4. git commit
  5. git log
  6. git diff & git show
  7. Mengenal .gitignore

1. git init — Memulai Repository

Perintah ini mengubah folder biasa menjadi repository Git, dengan membuat folder tersembunyi .git yang menyimpan seluruh riwayat.

bash 
mkdir proyek-latihan
cd proyek-latihan
git init
⚠️
Jangan double-init

Jalankan git init hanya sekali per proyek, di folder paling atas (root). Jangan jalankan di dalam folder yang sudah ada di dalam repo lain.

2. git status — Cek Kondisi Terkini

Perintah paling sering dipakai. Menampilkan file mana yang berubah, mana yang sudah di-stage, dan mana yang belum dilacak sama sekali.

bash 
git status

Biasakan menjalankan git status sebelum dan sesudah hampir setiap perintah lain — ini "kompas" kalian selama belajar Git.

📌
Tiga status setiap file di Git

Modified — file sudah diubah, tapi belum ditandai untuk disimpan. Staged — sudah ditandai lewat git add, siap masuk ke riwayat permanen. Committed — sudah resmi tersimpan lewat git commit. Analoginya: Modified itu barang belanjaan yang masih di rak, Staged itu sudah masuk keranjang, dan Committed itu sudah dibayar di kasir — resmi jadi milik kalian, tercatat di struk.

3. git add — Staging Perubahan

Memindahkan perubahan dari working directory ke staging area — memilih apa saja yang akan masuk ke commit berikutnya.

bash 
git add nama-file.txt      # stage satu file
git add folder/            # stage seluruh isi folder
git add .                  # stage semua perubahan di direktori saat ini
git add -p                 # stage sebagian (per potongan/hunk) dari sebuah file

4. git commit — Menyimpan Snapshot

Mengunci perubahan yang sudah di-stage menjadi satu titik riwayat permanen, lengkap dengan pesan penjelas.

bash 
git commit -m "Tambah halaman kontak"

# shortcut: stage semua file yang sudah dilacak + commit sekaligus
git commit -am "Perbaiki bug validasi form"
💡
Pesan commit yang baik

Tulis dalam kalimat perintah, jelas, dan spesifik: "Tambah validasi email di form pendaftaran" jauh lebih berguna daripada "update" atau "fix bug". Detail konvensi ada di Modul 7.

5. git log — Melihat Riwayat

bash 
git log                          # riwayat lengkap
git log --oneline                # ringkas, satu baris per commit
git log --oneline --graph --all  # + visualisasi branch dalam bentuk grafik

Setiap commit punya ID unik (hash) seperti a1b2c3d — ini yang dipakai untuk merujuk commit tertentu di perintah-perintah lain.

6. git diff & git show — Melihat Perubahan

bash 
git diff              # perubahan yang belum di-stage
git diff --staged     # perubahan yang sudah di-stage, belum di-commit
git show a1b2c3d       # detail lengkap satu commit tertentu

7. Mengenal .gitignore

Tidak semua file perlu dilacak Git — misalnya file dependency (node_modules/), file environment berisi kredensial (.env), atau file sistem (.DS_Store). Buat file bernama .gitignore di root proyek:

.gitignore 
node_modules/
.env
.DS_Store
*.log

Pembahasan lebih lengkap ada di Modul 7.

Rangkuman Siklus Harian

bash — siklus kerja harian 
git status              # 1. cek apa yang berubah
git add .               # 2. pilih perubahan yang mau disimpan
git status              # 3. konfirmasi ulang (kebiasaan baik)
git commit -m "pesan"   # 4. simpan sebagai snapshot
git log --oneline       # 5. lihat riwayatnya
