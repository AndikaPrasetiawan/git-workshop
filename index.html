Konsep Dasar

Apa Itu Git?

Git adalah sistem kontrol versi (version control system) terdistribusi yang mencatat setiap perubahan pada kode secara lokal, sehingga tim dapat berkolaborasi tanpa saling menimpa pekerjaan.

Sebelum Git

Berbagi kode lewat email & folder "final_v2"

Developer saling mengirim file zip, menamai file dengan "final", "final_fix", "final_beneran" — riwayat perubahan hilang dan konflik sulit dilacak.

2005

Linus Torvalds menciptakan Git

Dibuat untuk mengelola pengembangan kernel Linux yang melibatkan ribuan kontributor, butuh sistem cepat, terdistribusi, dan tahan konflik.

Sekarang

Standar industri version control

Hampir semua perusahaan teknologi menggunakan Git — dari startup kecil hingga perusahaan raksasa — sebagai fondasi kolaborasi kode.

Masalah Sebelum Ada Git

Tidak ada riwayat perubahan, sulit kembali ke versi sebelumnya, dan kerja tim rawan saling menimpa file satu sama lain.

Mengapa Git Dibuat

Untuk mencatat setiap perubahan secara otomatis, memungkinkan banyak orang bekerja paralel lewat branch, lalu menggabungkan hasilnya dengan aman.

Manfaat

Kenapa Perlu Belajar Git?

Enam alasan Git menjadi keahlian wajib bagi siapa pun yang menulis kode.

Version Control

Setiap perubahan tersimpan sebagai riwayat, sehingga kamu bisa kembali ke versi sebelumnya kapan saja.

Kolaborasi

Banyak orang bisa mengerjakan proyek yang sama secara bersamaan tanpa saling menimpa pekerjaan.

Riwayat Lengkap

Setiap commit menyimpan pesan, waktu, dan siapa penulisnya — jejak audit yang sangat berguna.

Backup Aman

Kode tersimpan di repository lokal maupun remote seperti GitHub, sehingga risiko kehilangan data jauh berkurang.

Branching

Kembangkan fitur baru di cabang terpisah tanpa mengganggu kode utama yang sudah stabil.

Kerja Tim yang Rapi

Alur pull request dan code review membuat kolaborasi tim jadi lebih terstruktur dan berkualitas.

Alur Kerja

Git Workflow

Empat tahap perjalanan sebuah file, dari komputer lokal hingga tersimpan di GitHub.

01

Working Directory

Folder proyek tempat kamu menulis dan mengubah file.

02

Staging Area

Area sementara untuk memilih perubahan mana yang akan disertakan pada commit berikutnya.

03

Local Repository

Riwayat commit tersimpan permanen di folder .git pada komputermu.

04

Remote Repository

Server seperti GitHub tempat kode dibagikan ke tim melalui git push.

Perbandingan

Git vs GitHub vs GitLab vs Bitbucket

Git adalah alat, sedangkan GitHub, GitLab, dan Bitbucket adalah layanan hosting yang menggunakan Git.

Aspek Git GitHub GitLab Bitbucket
Jenis Software version control Layanan hosting repository Layanan hosting repository Layanan hosting repository
Dijalankan Lokal di komputer Di cloud (server GitHub) Cloud atau self-hosted Cloud atau self-hosted
Pemilik Open source, komunitas Microsoft GitLab Inc. Atlassian
Fitur unggulan Commit, branch, merge Pull request, Actions, Pages CI/CD bawaan, DevOps lengkap Integrasi Jira & Trello
Wajib online? Tidak Ya, untuk kolaborasi Ya, untuk kolaborasi Ya, untuk kolaborasi
Kebiasaan Baik

Best Practices Menggunakan Git

Delapan kebiasaan yang membuat riwayat proyekmu tetap rapi dan mudah dipahami tim.

Commit

Meaningful Commit

Tulis pesan commit yang menjelaskan "kenapa", bukan sekadar "apa".

Commit

Small Commit

Satu commit untuk satu perubahan logis agar mudah ditelusuri dan di-revert.

Commit

Frequent Commit

Commit sesering mungkin saat progres jelas, jangan menumpuk perubahan terlalu besar.

Branch

Branch per Feature

Buat branch terpisah untuk setiap fitur atau perbaikan agar main tetap stabil.

Sinkronisasi

Pull Before Push

Tarik perubahan terbaru dengan git pull sebelum mengirim perubahanmu.

Keamanan

Never Commit Secrets

Jangan pernah commit API key, password, atau file kredensial ke repository.

Commit

Write Good Messages

Gunakan format konsisten, misalnya fix: perbaiki validasi form login.

Konfigurasi

Use .gitignore

Abaikan file yang tidak perlu dilacak seperti node_modules/ atau file environment.

Referensi Cepat

Git Cheat Sheet

Rangkuman perintah paling sering dipakai, dikelompokkan berdasarkan fungsinya.

Repository

git initBuat repository baru
git clone <url>Salin repository
git statusCek status file

Branch

git branchLihat/buat branch
git switch <nama>Pindah branch
git merge <nama>Gabungkan branch

Remote

git remote -vLihat remote
git pushKirim commit
git fetchAmbil data terbaru

History

git logLihat riwayat commit
git diffLihat perbedaan file
git show <hash>Detail satu commit

Undo

git reset <file>Batalkan staging
git checkout -- .Buang perubahan lokal
git revert <hash>Batalkan commit dengan aman

Tag

git tagLihat semua tag
git tag v1.0Buat tag versi
git push --tagsKirim tag ke remote
Peta Belajar

Learning Roadmap

Urutan belajar yang disarankan agar pemahamanmu terhadap Git terbangun bertahap.

1. Git Basics

Instalasi, konfigurasi, init, add, commit, dan status.

2. Branch

Membuat, berpindah, dan mengelola branch untuk fitur terpisah.

3. Merge

Menggabungkan hasil kerja dari beberapa branch ke satu branch utama.

4. Conflict

Memahami dan menyelesaikan konflik saat dua perubahan bertabrakan.

5. GitHub

Menghubungkan repository lokal ke remote repository di GitHub.

6. Pull Request

Mengajukan perubahan untuk direview sebelum digabung ke branch utama.

7. Collaboration

Bekerja dalam tim dengan alur branching dan code review yang jelas.

8. CI/CD

Mengotomatisasi testing dan deployment setiap ada perubahan kode.
