Remote & GitHub

Di sinilah repo lokal kalian "naik ke cloud" — modul ini menghubungkan semua yang sudah dipelajari ke GitHub.

Di modul ini
  1. Membuat repository di GitHub
  2. Setup SSH key
  3. Menghubungkan repo lokal ke remote
  4. git clone
  5. git push, pull, fetch
  6. Bonus: GitHub Pages

1. Membuat Repository di GitHub

  1. Login ke github.com, klik tombol New (atau ikon + di pojok kanan atas → New repository).
  2. Isi Repository name, misalnya materi-workshop.
  3. Pilih Public (agar bisa diakses lewat GitHub Pages nanti) atau Private.
  4. Jangan centang "Add a README file" kalau kalian sudah punya proyek lokal — biar tidak ada riwayat yang bentrok saat push pertama.
  5. Klik Create repository. GitHub akan menampilkan halaman instruksi berisi URL repo kalian.

2. Setup SSH Key (Sekali Saja per Komputer)

SSH key memungkinkan komputer kalian terhubung ke GitHub tanpa memasukkan username/password setiap kali push. Ini langkah sekali setup yang berlaku untuk semua repo di komputer tersebut.

bash 
# 1. Buat key baru (ganti email dengan email GitHub kalian)
ssh-keygen -t ed25519 -C "email@kamu.com"
# tekan Enter untuk semua pertanyaan (pakai lokasi & pengaturan default)

# 2. Jalankan ssh-agent lalu tambahkan key
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# 3. Salin public key ke clipboard
cat ~/.ssh/id_ed25519.pub
# copy seluruh output yang muncul
  1. Buka GitHub → foto profil → Settings → SSH and GPG keys → New SSH key.
  2. Tempel public key yang tadi disalin, beri judul (misalnya "Laptop Workshop"), lalu Add SSH key.
  3. Tes koneksinya:
bash 
ssh -T git@github.com
# kalau berhasil: "Hi username! You've successfully authenticated..."
📌
Alternatif: HTTPS + Personal Access Token

Kalau setup SSH menyulitkan di jaringan workshop, kalian bisa memakai URL HTTPS dan login dengan Personal Access Token (dibuat lewat Settings → Developer settings → Personal access tokens) sebagai pengganti password.

3. Menghubungkan Repo Lokal ke Remote

Kalau kalian sudah punya proyek lokal (dari Modul 3), sambungkan ke repo GitHub yang baru dibuat:

bash 
git remote add origin git@github.com:username/nama-repo.git
git branch -M main
git push -u origin main

origin adalah nama alias standar untuk remote utama. Flag -u menghubungkan branch lokal ke branch remote, jadi selanjutnya cukup git push tanpa argumen tambahan.

4. git clone — Mengunduh Repo yang Sudah Ada

Kalau repo sudah ada di GitHub (punya orang lain, atau punya kalian dari komputer lain), gunakan clone untuk mengunduh seluruh proyek beserta riwayatnya:

bash 
git clone git@github.com:username/nama-repo.git
cd nama-repo

5. push, pull, fetch

bash 
git push                   # kirim commit lokal ke remote
git push origin nama-branch  # push branch tertentu ke remote

git fetch                  # unduh perubahan remote TANPA menggabungkan ke kerja lokal
git pull                   # unduh + otomatis merge ke branch lokal (setara fetch + merge)
💡
Kapan pakai fetch vs pull?

fetch aman dipakai kapan saja — hanya melihat apa yang berubah di remote tanpa menyentuh kerja kalian. pull langsung menggabungkan, jadi pastikan pekerjaan lokal sudah di-commit dulu supaya tidak bentrok.

Bonus: Hosting Materi Ini dengan GitHub Pages

Karena materi workshop ini murni HTML/CSS/JS statis, kalian bisa langsung meng-hostingnya gratis lewat GitHub Pages — tanpa server, tanpa build step:

  1. Push folder materi-workshop ini ke sebuah repo GitHub (lihat langkah 1–3 di atas).
  2. Buka repo di GitHub → tab Settings → menu Pages di sidebar kiri.
  3. Di bagian Build and deployment, pilih source Deploy from a branch.
  4. Pilih branch main dan folder / (root), lalu klik Save.
  5. Tunggu 1–2 menit, situs akan tersedia di https://username.github.io/nama-repo/.
